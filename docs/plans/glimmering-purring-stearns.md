# Obsidian MCP-сервер

## Context

Цель — учебная: разобраться, как устроены MCP-серверы, на практике. Нужен лёгкий, но реально полезный сервер поверх уже существующего Obsidian vault пользователя, который даст Claude доступ к заметкам прямо в разговоре. Путь до vault не хардкодится — берётся из переменной окружения, чтобы сервер был портируемым и не завязан на конкретную машину.

`fastmcp` уже добавлен в зависимости проекта (`pyproject.toml`).

## Дизайн

Новая директория `obsidian-mcp/` (по аналогии с `local-llm-lab/` — плоский модуль, без пакета) с одним файлом `server.py`.

**Конфигурация**
- Путь к vault — `OBSIDIAN_VAULT_PATH` (env var), резолвится в абсолютный путь при старте.
- Если переменная не задана или путь не существует/не директория — сервер падает сразу с понятной ошибкой (fail fast, без силент-фоллбэков).

**Безопасность (единственная реальная граница входа — путь от клиента)**
- Общий хелпер `_resolve(rel_path: str) -> Path`: джойнит vault root + относительный путь, резолвит `..`/симлинки и проверяет, что результат остаётся внутри vault root. Иначе — ошибка. Используется везде, где путь приходит от вызывающего.

**Tools** (`@mcp.tool`)
1. `list_notes(folder: str = "") -> list[str]` — список относительных путей `.md`-файлов под vault, опционально отфильтрованных по префиксу папки.
2. `search_notes(query: str, limit: int = 20) -> list[dict]` — регистронезависимый поиск подстроки по именам файлов и содержимому; возвращает `{path, line, snippet}` по каждому совпадению, максимум `limit` результатов.
3. `read_note(path: str) -> str` — читает и возвращает сырой markdown заметки по относительному пути (через `_resolve`).

**Resource**
- `@mcp.resource("note://{path}")` — та же логика чтения, что и `read_note`, но как resource-шаблон — чтобы на практике увидеть разницу между tool и resource в MCP.

Никакого write/append функционала в v1 — держим сервер read-only и простым; при необходимости расширяется отдельной задачей.

**Регистрация сервера**
Добавить в `.mcp.json` (по образцу существующих stdio-серверов):
```json
"obsidian": {
  "type": "stdio",
  "command": "uv",
  "args": ["run", "python", "obsidian-mcp/server.py"],
  "env": { "OBSIDIAN_VAULT_PATH": "${OBSIDIAN_VAULT_PATH}" }
}
```
Пользователь сам прописывает `OBSIDIAN_VAULT_PATH` в своём окружении.

**Документация**
Добавить строку про `obsidian-mcp/` в `docs/INDEX.md` (по образцу существующих пунктов).

## Реализация

- Делегировать код `backend-dev` (это Python/бэкенд-код, серверная логика).
- После реализации — обязательный проход `code-reviewer`, цикл до отсутствия замечаний.

## Верификация

- `uv run fastmcp dev obsidian-mcp/server.py` — поднимает MCP Inspector, вручную вызвать `list_notes`, `search_notes`, `read_note`, проверить чтение resource `note://...` и что путь с `../` за пределы vault отклоняется.
- Убедиться, что сервер падает с понятной ошибкой, если `OBSIDIAN_VAULT_PATH` не задан.
